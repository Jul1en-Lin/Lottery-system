# 管理员登录页重设计文档

## 1. 目标

当前登录页虽然已经具备管理员双模式登录能力，但视觉焦点分散，左侧展示区和右侧表单区的层级不够清晰，未能充分复用 demo 的核心优势。本次重设计目标如下：

1. 参考 login-demo.html 的两栏式海报布局，恢复左侧展示区与右侧登录区的强对比关系。
2. 左侧不再承载大段说明文案，改为品牌 + 角色舞台 + 轻量底部链接，突出展示感。
3. 右侧表单区采用更居中的登录面板，弱化杂讯，提升高级感与可读性。
4. 保留现有业务逻辑：管理员密码登录、管理员邮箱验证码登录、登录方式切换动画、管理员注册跳转。
5. 仅重做登录页，不破坏注册页与找回密码页的现有结构。

## 2. Demo 抽取出的设计语言

参考 login-demo.html，核心设计特征不是“卡通角色”本身，而是以下几项组合：

1. 左侧 Hero 是完整的海报式展示区，内容结构为“品牌头部 / 舞台主体 / 轻量底部链接”。
2. 左侧主体高度充足，角色区被放在视觉中心附近，而不是被文字切碎。
3. 右侧表单区不是厚重卡片，而是干净的留白面板，依赖排版、间距和弱阴影建立品质感。
4. 登录标题区采用居中布局，表单交互与主按钮形成单一视觉路径。
5. 背景层次以柔和径向光晕、网格和浅雾化阴影构成，而不是大量辅助信息。

## 3. 布局方案

### 3.1 左侧展示区

采用三段结构：

1. 顶部：品牌标识，仅保留 Logo 与品牌名称。
2. 中部：角色舞台，作为整页第一视觉焦点，占据 Hero 的主要高度。
3. 底部：仅保留隐私政策、服务条款两个轻量链接，避免噪音。

左侧不再出现说明性标题、徽标文案和辅助段落，避免信息与角色场景竞争。

### 3.2 右侧登录区

右侧维持管理员登录功能，但视觉上改成更贴近 demo 的“居中面板”方案：

1. 标题居中。
2. 表单宽度控制在舒适阅读区间。
3. 背景增加很弱的径向光晕，提升层次。
4. 保留记住设备、忘记密码和管理员注册入口，但保持低干扰。

## 4. 视觉规范

### 4.1 左侧

1. 沿用现有灰阶渐变背景，但增加登录页专属的柔光层。
2. 角色场景回归居中展示，重新分配前后景尺寸，形成主次关系。
3. 舞台底座与阴影扩大，增强“展台感”。

### 4.2 右侧

1. 表单区背景保持白色，但用轻微径向渐变和更克制的阴影提升质量。
2. 标题区参考 demo，采用“欢迎回来 + 辅助说明”的短结构。
3. 登录方式切换条继续保留，但作为表单内部的精细交互，不与主版式竞争。

## 5. 交互策略

1. 登录方式切换保留现有滑块动画。
2. 密码登录与邮箱验证码登录继续共用一个表单框架。
3. 邮箱验证码登录只保留输入框内发送验证码按钮，不再提供重复入口。
4. 左侧角色继续响应鼠标移动与输入状态，但强度控制在辅助层，不抢占表单注意力。

## 6. 实现映射

本次代码改造对应如下：

1. login.html：增加登录页专属结构类，重排左侧 Hero 与右侧表单容器。
2. auth.css：新增登录页专属布局覆盖，避免影响 signup 和 forgot-password。
3. auth.js：保留现有登录逻辑，仅复用已有切换和场景动效，不改接口调用。

## 7. 验收标准

1. 左侧第一眼应先看到品牌与角色场景，而不是大段文字。
2. 右侧表单应在视觉中心附近，标题与表单形成单一路径。
3. 登录方式切换时不跳动、不撑高页面。
4. 密码登录和邮箱验证码登录的辅助区布局一致。
5. 页面桌面端更接近 demo 的海报感，移动端仍按当前响应式规则自动折叠为单栏。
