# 新闻纸复古风格前端重构设计文档

## 概述

将 Lottery-system 前端从现代 SaaS 风格重构为新闻纸复古风格，打造"一张有温度的旧报纸"视觉体验。

## 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 重构范围 | 统一重构所有页面 | 保持风格一致性 |
| 风格方向 | 复古怀旧 - 纸质质感 | 与抽奖主题契合 |
| 质感细节 | 新闻纸纹理、手写字体、邮戳印章、磨损边缘 | 营造真实纸张感 |
| 配色方案 | 新闻纸系 | 泛黄白纸底色、黑色油墨、红色印章强调 |
| 字体风格 | 纯手写风格 | 中文手写体 + 英文打字机字体 |
| 技术框架 | Vue 3 + Element Plus | 组件化开发、生态成熟 |
| 构建工具 | Vite + JavaScript | 快速构建、配置简单 |
| 部署方式 | 集成部署 | 构建产物放入 Spring Boot static 目录 |
| 动画效果 | 丰富动画 | 翻页、刮奖、印章等沉浸式交互 |
| 端适配 | 仅 PC 端 | 专注桌面体验 |
| 团队协作 | Agent team 按模块分工 | 并行开发提高效率 |

## 视觉规范

### 配色系统

```
主背景:     #F5F0E1 (泛黄新闻纸)
次背景:     #EBE6D7 (深一度纸张)
主文字:     #1A1A1A (黑色油墨)
次文字:     #4A4A4A (灰色油墨)
强调色:     #C41E3A (邮戳红)
辅助强调:   #2B5F8E (蓝墨水)
边框色:     #8B8B8B (灰调边框)
分割线:     #C9C4B8 (淡灰线)
```

### 字体规范

```
中文字体:
  - 主标题: 站酷快乐体 / 沐瑶软笔手写体
  - 正文: 站酷快乐体 (较大字号)

英文字体:
  - 主字体: Special Elite (打字机风格)
  - 备选: Courier New, Courier

字体大小:
  - 头版标题: 48-72px
  - 副标题: 24-32px
  - 正文: 16-18px
  - 辅助文字: 12-14px
```

### 质感素材

- 纸张纹理：SVG 噪点图案叠加
- 磨损边缘：CSS border-image 或 SVG 遮罩
- 印章效果：旋转 -5° ~ 5°，添加不规则边框
- 印刷网点：径向渐变模拟网点效果

```scss
.stamp-appear {

@keyframes stamp-down {
  0% {

    opacity: 0;
  }
  50% {
    transform: scale(0.9) rotate(-3deg);
  }
  100% {
    transform: scale(1) rotate(-5deg);
    opacity: 1;
  }
}
```

## Agent Team 分工方案

> **并发限制：** API 限速，采用顺序执行策略，每次只派一个 Agent。

### 阶段一：项目初始化（当前会话直接执行）

任务：
- 创建 Vue 3 + Vite 项目
- 配置 Element Plus
- 配置路由和状态管理
- 设置构建输出目录为 `../src/main/resources/static`
- 创建基础目录结构
- 配置 API 代理和请求封装

### 阶段二：模块顺序开发

| 顺序 | Agent | 任务内容 |
|------|-------|----------|
| 1 | Agent 1 | 组件库：PaperCard、Stamp、InkButton、NewspaperTitle、Divider、纹理素材 |
| 2 | Agent 2 | 页面框架：NewspaperLayout、NavBar、Footer、路由配置完善 |
| 3 | Agent 3 | 动画系统：翻页效果、刮奖组件、印章动画、微交互动画库 |
| 4 | Agent 4 | 认证页面：LoginView、SignupView、ForgotPasswordView |
| 5 | Agent 5 | 业务页面：HomeView、ActivityListView、ActivityDetailView |
| 6 | Agent 6 | 用户相关：UserCenterView、PrizeRecordView、AdminView |

### 阶段三：集成部署（当前会话直接执行）

任务：
- 整合所有模块
- 配置生产构建
- 测试构建产物
- 验证 API 交互
- 编写部署文档

## 构建配置

### vite.config.js
```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  },
  build: {
    outDir: '../src/main/resources/static',
    emptyOutDir: true,
    rollupOptions: {
      output: {
        assetFileNames: 'assets/[name].[ext]'
      }
    }
  },
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
})
```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 手写字体加载慢 | 首屏体验 | 字体子集化、预加载 |
| 动画性能问题 | 用户体验 | 使用 CSS 动画、减少重绘 |
| Element Plus 样式冲突 | 开发效率 | 使用 CSS Modules、深度定制 |
| 构建产物过大 | 加载速度 | 代码分割、按需引入 |
